1. Promise vs Reality

Hạng mụcNội dung
Product Promise Moni hỗ trợ người dùng tìm kiếm, tra cứu và thanh toán hóa đơn bằng ngôn ngữ tự nhiên.
Target User Người dùng MoMo muốn tìm hóa đơn nhanh mà không cần đi qua nhiều màn hình.
Expected Task Người dùng nhập: "Đưa hóa đơn điện nước". Hệ thống nhận diện đúng loại hóa đơn và cho phép thanh toán.
Reality Moni trả về hóa đơn điện thoại thay vì hóa đơn điện hoặc nước.

2. Evidence

  • Prompt thử nghiệm: "Đưa hóa đơn điện nước"
  • Kết quả quan sát: Moni trả về hóa đơn điện thoại.
  • Không xuất hiện màn hình xác nhận lại intent.
  • Không có nút sửa kết quả hoặc chọn lại loại hóa đơn.
  • Screenshot thực tế cần đính kèm khi nộp bài.

3. Finding → Product Decision

Khi user yêu cầu xem hóa đơn điện nước,
AI hiểu sai intent và trả về hóa đơn điện thoại.
Hậu quả là user nhận sai thông tin và workflow thanh toán bị đứt gãy.
Lỗi thuộc layer Intent + UX Recovery.
Nên sửa bằng Clarification Flow, Correction Flow và lưu Correction Log để cải thiện mô hình.

5. 4 Paths Analysis

Happy Path
AI nhận diện đúng loại hóa đơn → hiển thị hóa đơn → thanh toán ngay.
Low-confidence Path
Nếu confidence thấp (ví dụ dưới ngưỡng đề xuất 70%), hệ thống hỏi lại: [Điện] [Nước] [Điện thoại]
Failure Path
Nếu AI trả sai kết quả, hiển thị nút: "Chọn lại loại hóa đơn" hoặc "Báo lỗi".
Correction Path
Thông tin user sửa được lưu vào Correction Log → Eval Dataset → cải thiện phiên bản tiếp theo.

7. SPEC Change

  • Thêm Clarification Flow khi intent không rõ ràng.
  • Thêm Correction Chips dưới mỗi kết quả.
  • Thêm cơ chế lưu Correction Log.
  • Định nghĩa ngưỡng confidence để kích hoạt hỏi lại.
  • Bổ sung chỉ số User Correction Rate.

8. KPI Đề Xuất

  • Intent Accuracy > 95%
  • User Correction Rate
  • Chip Click Through Rate
  • Average Resolution Turns
  • Correction Log Coverage
